## 🚀 快速開始：兩階段設置

## 第一階段：建立資料收集系統 📊

### 步驟 1：建立 Google Apps Script 專案

#### 1.1 開啟 Google Apps Script
- 前往：[script.google.com](https://script.google.com/)
- 用你的 Google 帳號登入

#### 1.2 建立新專案
- 點擊「新增專案」
- 重新命名為「YouTube 趨勢追蹤系統」

#### 1.3 設置程式碼
- 複製專案中的 `src/youtube-search-api.gs` 檔案內容
- 貼上到 Google Apps Script 編輯器中

### 步驟 2：啟用 YouTube API

#### 2.1 在 Apps Script 中啟用
1. 點擊左側「服務」（Services）
2. 點擊「新增服務」➕
3. 搜尋並新增：`YouTube Data API v3`

#### 2.2 在 Google Cloud 中啟用
1. 前往：[console.cloud.google.com](https://console.cloud.google.com/)
2. 選擇專案 → API 和服務 → 程式庫
3. 搜尋 `YouTube Data API v3` → 啟用

### 步驟 3：執行初始化

#### 3.1 首次授權
- 選擇 `setupSystem` 函數
- 點擊執行，完成 Google 授權流程

#### 3.2 開始資料收集
- 執行 `dailyYouTubeTracking()` 開始收集資料
- 等待 3-5 分鐘完成首次執行

## 第二階段：使用 Web 分析工具 🌐

### 工具 1：影片搜尋分析 📺

**檔案位置**：`src/youtube-search.html`

#### 🎯 主要功能
- **響應式影片網格**：5 列精美卡片展示
- **多維度篩選**：地區、類型、關鍵字、觀看數範圍
- **即時搜尋**：標題和 hashtags 全文搜尋
- **統計顯示**：觀看數、按讚數、留言數、影片時長

#### 📱 使用方法
1. 在瀏覽器中開啟 `src/youtube-search.html`
2. 系統自動載入最新資料
3. 使用頂部篩選器進行數據篩選：
   - **地區篩選**：美國、印度、台灣、巴西、印尼、墨西哥
   - **類型篩選**：一般影片、Shorts 短影片
   - **觀看數篩選**：設定最小/最大觀看數範圍
   - **關鍵字搜尋**：輸入關鍵字搜尋標題和標籤
4. 點擊影片縮圖直接在 YouTube 觀看

### 工具 2：Hashtag 統計分析 📊

**檔案位置**：`src/hashtag-analytics.html`

#### 🎯 主要功能
- **Chart.js 視覺化**：圓餅圖、長條圖、環圈圖可切換
- **雙重排行榜**：使用頻率排行 + 總觀看數排行
- **多維度篩選**：時間範圍、地區、類型、排序方式
- **詳細數據表格**：完整統計數據和百分比

#### 📈 使用方法
1. 點擊頁面頂部「Hashtag 分析」按鈕
2. 選擇篩選條件：
   - **時間範圍**：最近 7 天、30 天、全部
   - **地區選擇**：特定地區或全部地區
   - **影片類型**：Videos、Shorts 或全部
   - **排序方式**：頻率、觀看數、按讚數等
3. 切換圖表類型：
   - **圓餅圖**：顯示 hashtag 使用比例
   - **長條圖**：顯示數值排行
   - **環圈圖**：顯示雙層數據關係
4. 查看詳細數據表格了解完整統計

### 工具 3：標題文字雲分析 ☁️

**檔案位置**：`src/title-wordcloud.html`

#### 🎯 主要功能
- **智能分詞**：支援中英文混合文本分析
- **動態文字雲**：字體大小和顏色反映重要程度
- **停用詞過濾**：自動過濾無意義詞彙
- **互動式設計**：點擊關鍵詞查看相關影片

#### ☁️ 使用方法
1. 點擊頁面頂部「文字雲分析」按鈕
2. 設定分析參數：
   - **時間範圍**：選擇分析的時間區間
   - **地區篩選**：選擇特定地區或全部
   - **最小字數**：設定顯示關鍵詞的最小長度
3. 查看文字雲結果：
   - **字體大小**：反映詞彙出現頻率
   - **顏色深淺**：反映相關影片觀看數
   - **點擊互動**：點擊詞彙查看包含該詞的所有影片
4. 參考排行榜：
   - **頻率排行**：最常出現的關鍵詞
   - **觀看數排行**：帶來最高觀看數的關鍵詞

## 🔧 進階功能和管理

### 系統狀態檢查
執行這些函數來管理你的系統：
- `checkSystemStatus()` - 檢查整體狀態
- `setupSystem()` - 重新初始化系統
- `dailyYouTubeTracking()` - 手動執行資料收集
- `setupDailyTrigger()` - 設定每日自動執行
- `removeDailyTrigger()` - 停止自動執行

### Web API 功能
如需讓其他應用程式存取資料：
1. 在 Google Apps Script 中部署為「網路應用程式」
2. 設定存取權限為「任何人」
3. 使用產生的 URL 存取 JSON 格式資料
4. 支援參數：`?regionCode=TW&days=3&max=10`

### 自定義設定
編輯程式中的常量來調整系統：
```javascript
const DAILY_SEARCH_DAYS = 5;        // 搜尋過去幾天
const RANKING_LIMIT = 20;           // 每個地區保留幾名
const SHORTS_DURATION_LIMIT = 60;   // Shorts 長度閾值（秒）
```

## 🔧 疑難排解

### 資料收集問題
**❌ 「YouTube is not defined」**
- 確保已啟用 YouTube Data API v3 服務

**❌ 「配額已用盡」**
- YouTube API 每日免費額度用完，隔天再試

**❌ 「權限不足」**
- 重新執行 `setupSystem()` 完成授權

### Web 工具問題
**❌ 網頁顯示「載入資料中...」**
- 檢查 Google Apps Script API 是否部署為網路應用程式
- 確認資料來源 URL 是否正確

**❌ 圖表無法顯示**
- 確保瀏覽器支援 JavaScript
- 檢查網路連接是否正常

**❌ 文字雲空白**
- 確認有足夠的標題資料
- 調低「最小字數」設定
